** telnet による demoモード

java ディレクトリに移動し、サーバを起動します。

$ java -cp bin dhm.Server -p 8000 -d

-p は listenするポート、-d はデバッグモードです。
そのままもう一枚terminal を開き、java ディレクトリに移動の上、以下を実行して下さい。

$ sh demo.sh

サーバに A B C D の4人のロボットが接続し、待機状態になります。
さらに、`telnet localhost 8000` が実行され、ユーザ名の入力を求められます。

※telnetで参加する際、手持ちのカードを覚えておくのが難しいため、ヘルプ機能が用意されています。
- 名前の末尾が "?" の場合、自TURN時に手持ちのカード(c)やしばり(b)、革命(r)、直前のカード(p)が通知されます。
- 名前の末尾が "!" の場合、さらに上記の情報が色つきで通知されます。

名前を入力するとゲームが開始されるので、「DHMPについて」を参照の上カードを出して下さい。
カードはスペース区切りまたはカンマ区切りのどちらでも指定できます。



** ロボット能力測定モード

java ディレクトリに移動し、サーバを起動します。

$ java -cp bin dhm.Server -p 8000 -n 1000

-n でゲーム回数を指定します。
そのままもう一枚terminal を開き、java ディレクトリに移動の上、以下を実行して下さい。

$ sh bench.sh

測定が開始され、サーバ側に結果が表示されます。
自作ロボットが完成したら、bench.sh に組み込んで実行して下さい。
